# Phase 07.2: Motion Planning Problem Analysis — Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Diagnose AND fix — inline, within this phase — the three motion-planning flakiness items documented in `07-PHASE6-REGRESSION.md §Follow-up`:

1. **FC-1 · No geometric IK** — `grasp_move` fails with "no geometric IK" for lego blocks spawned near the SO-ARM101 workspace edge.
2. **FC-2 · OMPL error −2 (grasp_home)** — `grasp_home` fails with MoveIt error `-2` when called from the post-release arm configuration.
3. **FC-3 · Out-of-reach block spawns** — `randomize_object_poses` (MCP tool) samples xyz that lands outside the reachable workspace, upstream-causing FC-1 at a measurable rate.

The phase delivers (a) a root-cause diagnosis per failure class grounded in workspace / configuration-space math, backed by N=10 instrumented trials per class, and (b) landed fixes with regression evidence. Phase 10 no longer owns these three items.

New capabilities (drop orientation, trajectory-based grasp, jerk reduction, camera-guided drop) stay in Phase 10.
</domain>

<decisions>
## Implementation Decisions

### Investigation Design

- **D-01 · Evidence method:** Reachable-workspace + C-space analysis paired with N=10 instrumented trials per failure class. Per class, deliver a math artifact (reachable set overlay for FC-1/FC-3; C-space cluster map + planning-scene audit for FC-2) and an empirical rate estimate.
- **D-02 · Trial rigor:** N=10 random-seed trials per class. No deterministic repro harness (skipped to save time — revisit only if N=10 variance is unusable).
- **D-03 · Environment:** Isaac Sim only. All three classes are sim-observable with full state observability; real-hardware sampling is out of scope for 07.2.
- **D-04 · Scope mode:** Analysis + inline fix. 07.2 owns all three fixes. No routing to 8/9/10.

### Failure Class FC-1 · No geometric IK at workspace edge

- **D-05 · Analytical step:** Compute the SO-ARM101 reachable set R ⊂ SE(3) by sweeping joint limits × `grip_angle` × `wrist_roll`. Output: a reachable-workspace artifact (point cloud or voxel grid in base frame).
- **D-06 · Empirical step:** For each N=10 failing trial, log the block spawn xyz, the grip_angle/wrist_roll used by `_solve_grasp_ik`, and the geometric_ik return code. Confirm failing poses lie outside R.
- **D-07 · Fix class:** Upstream filter in the MCP `randomize_object_poses` tool (see FC-3). If a downstream failure survives the filter, handle at `_solve_grasp_ik` with explicit "out-of-reach" response rather than `None`.
- **D-08 · Claude's Discretion:** Which representation for R (occupancy grid vs convex hull vs sampled point cloud). Roboticist default: voxel grid at ~5mm resolution in the cylindrical workspace annulus, cheap to intersect against spawn samples.

### Failure Class FC-2 · OMPL error −2 from post-release configuration

- **D-09 · Analytical step:** Dump joint positions at the moment `grasp_home` is called across all N=10 trials (failing + passing). Cluster in C-space (6D joint space, or task-relevant 5D). Identify the failing sub-region and its geometric relationship to the canonical passing start set.
- **D-10 · Planning-scene audit:** At `grasp_home` call time, dump the MoveIt planning scene JSON (collision objects, attached objects, ACM entries). Check for stale cup objects, orphaned attached block, or padding artifacts that shrink the free space disproportionately near the post-release config.
- **D-11 · Fix candidates (pick after evidence lands):** (i) inject a short warmup/neutralize move to exit the failing C-space region before calling `grasp_home`; (ii) bump OMPL budget for `grasp_home` specifically (10s → 30s); (iii) fix a planning-scene bug if the audit surfaces one. D-02 trials tell us which.
- **D-12 · Claude's Discretion:** Choice between warmup-move vs budget-bump vs scene-fix is DATA-DRIVEN — do not pre-commit. The 07.2 PLAN will branch on the evidence.

### Failure Class FC-3 · Out-of-reach block spawns

- **D-13 · Analytical step:** Overlay the `randomize_object_poses` sample distribution on R (from D-05). Quantify out-of-reach fraction. This directly bounds the upper limit of FC-1's rate.
- **D-14 · Fix:** Add a reachability filter to the MCP `randomize_object_poses` tool — reject samples outside R before spawning. Tighten the spawn box if the rejection rate is high (>20%), to keep trials efficient.
- **D-15 · Regression:** After landing D-14, re-run N=10 to verify FC-1 rate drops to the residual (non-spawn-caused) level and FC-3 rate → 0.

### Output Artifacts

- **D-16 · FINDINGS.md** in phase dir — failure taxonomy table, rate estimates (pre-fix + post-fix), analytical artifacts referenced (paths to R voxel grid, C-space cluster map, planning-scene dumps).
- **D-17 · Code changes live in** `~/Projects/Exploring-VLAs/vla_SO-ARM101` (control_gui.py for IK-layer fixes) and this repo's `exts/soarm101-dt/so_arm101_dt/extension.py` (MCP randomize tool filter).
- **D-18 · Success criteria (updated for fix-inline scope):**
  1. FINDINGS.md published with per-class root-cause statement grounded in analytical evidence.
  2. N=10 trial rates captured pre-fix and post-fix for each class.
  3. Reachability filter landed in `randomize_object_poses`; FC-3 rate → 0 post-fix.
  4. FC-1 residual rate (post-filter) measured and documented — if non-zero, explained.
  5. FC-2 fix landed (warmup / budget / scene fix as evidence dictates) with N=10 post-fix verification.
  6. No regression in the Phase 6 canonical pick-drop cycle — 5/5 consecutive clean passes post-fix under the same random-spawn conditions.

### Claude's Discretion

- R's representation (D-08).
- FC-2 fix selection branch (D-12).
- Whether to add a fixed-seed repro harness if N=10 variance proves too noisy (skipped by default per D-02).
- Online references to consult during research (OMPL tuning for short-horizon plans in MoveIt 2 Humble; 5-DOF geometric IK with grip-angle parameterization; planning-scene sync anti-patterns).

</decisions>
