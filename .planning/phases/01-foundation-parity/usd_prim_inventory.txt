## Decision (recorded by Plan 05 Task 1)

Strategy: PER-FRAME-RAW-OVERRIDE
Rationale: 16 of 32 live frames are USD prims that exist but with underscore-flattened names (e.g., USD prim `/World/aic_unified_robot/gripper_hande_base_link` must publish as ROS frame_id `gripper/hande_base_link`). 2 entries are non-prim noise/synthesis: `aic_world` is a real Gazebo static-published synthesized frame with no driving prim (must be published as a static `world -> aic_world` identity edge), and `view_frames Result` is a .gv legend artifact (filter at consumer — NOT a real ROS frame, ignore in Plan 06 builder). USD sublayer renaming was rejected because (a) USD prim names cannot legally contain `/` (it is the path separator), so any `over` block trying `over "gripper/hande_base_link"` produces an invalid Sdf path; (b) even if `customData:frameId` were authored, ROS2PublishTransformTree in Isaac Sim 5.0 reads the prim's `name` (the `Sdf.Path` leaf), not custom metadata, so the rename would have no effect on the published frame_id. Therefore PER-FRAME-RAW-OVERRIDE is the ONLY workable strategy for these 16 + 1 frames; Plan 06's TF builder will read the override list above and emit one `ROS2PublishRawTransformTree` (or equivalently, populate per-frame `targetPrim` + `parentFrame`/`childFrame` overrides) per entry.
Affected prims: 17 (16 underscore-rename + 1 synthesized aic_world edge)
Implementing task: Task 3 SKIPPED (no sublayer authored). Plan 06 Task 1 (TF builder) reads this `[Override list for Plan 06 / sublayer]` section and creates per-frame Raw publishers. The 14 MATCH frames (base, base_link, base_link_inertia, flange, forearm_link, ft_frame, shoulder_link, tabletop, tool0, upper_arm_link, world, wrist_1_link, wrist_2_link, wrist_3_link) can be published via the standard ROS2PublishTransformTree default name mapping. `view_frames Result` is filtered out (legend artifact, not a real frame).
