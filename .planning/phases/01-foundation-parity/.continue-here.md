Re-probe before drawing any conclusions.

- [ ] **CONSTRAINT: USD `metersPerUnit` + `upAxis` mismatch silently breaks AddReference.** AIC's IsaacLab-canonical `*_visual.usd` files are mPU=0.01 + Y-up; the parent stage (unified robot USD + scene/aic.usd) is mPU=1.0 + Z-up. Isaac Sim does NOT auto-rescale on `AddReference` — geometry imports 100x smaller and rotated 90°. This is the root cause of the user-reported "major mismatch in object size". Structural mitigation: when adding any USD reference to extension.py's `AIC_OBJECTS` or any spawn-atom path, verify mPU/upAxis match parent stage via `pxr.UsdGeom.GetStageMetersPerUnit/GetStageUpAxis`. If mismatched, EITHER bake xformOp:scale + xformOp:rotateXYZ at the wrapper prim, OR re-author the source USD with parent-stage units, OR use a different USD variant. Never assume `*_visual.usd` is interchangeable with the original.

- [ ] **CONSTRAINT: byte-identical filename ≠ byte-identical content in vendored trees.** Plan 02 vendored AIC's `Intrinsic_assets/assets/<Capitalized>/` tree, which has BOTH the original USDs (e.g. `task_board_rigid.usd`, mPU=1.0+Z-up) AND `*_visual.usd` variants (mPU=0.01+Y-up) sitting side-by-side as different files. Plan 04 silently swapped paths to the wrong variant. Structural mitigation: when changing any USD path string in extension.py, run pxr inspection on both the old and new path to confirm equivalent properties (mPU, upAxis, defaultPrim, prim count, mesh count) BEFORE committing.
