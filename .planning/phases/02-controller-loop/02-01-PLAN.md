# AIC Custom Messages Setup for Isaac Sim 5.0

Phase 2's controller loop subscribes to `aic_control_interfaces/{JointMotionUpdate, MotionUpdate, ControllerState}` and publishes `ros_gz_interfaces/Contacts`. Neither package ships in the IsaacSim-ros_workspaces base build; both must be added to the Python 3.11 workspace and rebuilt.

## Why not use the AIC pixi env directly?

The `~/Documents/aic` repo's pixi env builds `aic_control_interfaces` against **Python 3.12**. Isaac Sim 5.0 runs **Python 3.11**. CPython C-extension `.so` files (which ROS message typesupport ships as) are NOT ABI-compatible across minor versions. Loading the pixi env's modules from Isaac Sim's Python 3.11 fails at the typesupport-`.so` import (verified 2026-05-03; even numpy fails to import from the pixi env under 3.11).

CONTEXT.md's D-05 originally proposed PYTHONPATH-linking the pixi env. **That approach is broken; this doc is the replacement.**

## Prerequisites

- Phase 1's IsaacSim-ros_workspaces is built and present at `~/IsaacSim-ros_workspaces/build_ws/humble/humble_ws/`. If not, complete `rclpy-setup.md` first.
- Docker installed and working (the build runs inside a Humble container).
- ~10-20 minutes for the rebuild.

# 2. Clone ros_gz (humble branch) and vendor ros_gz_interfaces
cd /tmp && rm -rf ros_gz && git clone -b humble https://github.com/gazebosim/ros_gz.git

      ~/IsaacSim-ros_workspaces/build_ws/humble/humble_ws/src/

# 3. Rebuild the workspace

```bash
#!/usr/bin/env bash
# scripts/build_aic_msgs.sh — Phase 2 D-05 fix: rebuild Python 3.11 workspace
#                              with aic_control_interfaces + ros_gz_interfaces.
# See exts/aic-dt/docs/aic-msgs-setup.md for full background.
set -euo pipefail

WS="$HOME/IsaacSim-ros_workspaces/build_ws/humble/humble_ws/src"
[[ -d "$WS" ]] || { echo "ERROR: workspace not found at $WS — run exts/aic-dt/docs/rclpy-setup.md first"; exit 1; }

# 1. Vendor aic_control_interfaces (idempotent — re-copy each time so source-of-truth tracks AIC)
SRC_AIC="$HOME/Documents/aic/aic_interfaces/aic_control_interfaces"
[[ -d "$SRC_AIC" ]] || { echo "ERROR: AIC source not found at $SRC_AIC"; exit 1; }
rm -rf "$WS/aic_control_interfaces"
cp -r "$SRC_AIC" "$WS/"
echo "Vendored aic_control_interfaces -> $WS/aic_control_interfaces"

# 2. Clone & vendor ros_gz_interfaces (idempotent — re-clone each time for freshness)
cd /tmp && rm -rf ros_gz
git clone -b humble https://github.com/gazebosim/ros_gz.git --depth 1
rm -rf "$WS/ros_gz_interfaces"
cp -r /tmp/ros_gz/ros_gz_interfaces "$WS/"
echo "Vendored ros_gz_interfaces -> $WS/ros_gz_interfaces"

# 3. Rebuild
cd "$HOME/IsaacSim-ros_workspaces" && bash build_ros.sh -d humble -v 22.04

# 4. Sanity-check 3.11 ABI
INSTALL="$HOME/IsaacSim-ros_workspaces/build_ws/humble/humble_ws/install/local/lib/python3.11/dist-packages"
[[ -f "$INSTALL/aic_control_interfaces/__init__.py" ]] || \
  { echo "ERROR: $INSTALL/aic_control_interfaces/__init__.py missing — workspace build failed"; exit 1; }
[[ -f "$INSTALL/ros_gz_interfaces/__init__.py" ]] || \
  { echo "ERROR: $INSTALL/ros_gz_interfaces/__init__.py missing — workspace build failed"; exit 1; }

# 5. Reject cpython-310 tagged .so (would mean wrong-Python build)
if ls "$INSTALL/aic_control_interfaces/"*.so 2>/dev/null | grep -q cpython-310; then
  echo "ERROR: aic_control_interfaces .so files tagged cpython-310 — workspace built against wrong Python"
  exit 1
fi

echo "OK: aic_control_interfaces + ros_gz_interfaces built for Python 3.11"
```

```bash
#!/usr/bin/env bash
# scripts/snapshot_aic_eval_offlimit.sh — Phase 2 D-10 + Open Q1 settlement.
# Captures: (a) full aic_controller-namespace topic list (Open Q1),
#           (b) /aic/gazebo/contacts/off_limit echoes during a CheatCode trial (D-10).
#
# Usage: ./snapshot_aic_eval_offlimit.sh [DEST]
#   DEST defaults to .planning/phases/02-controller-loop/snapshot
set -euo pipefail

DEST="${1:-.planning/phases/02-controller-loop/snapshot}"
mkdir -p "$DEST"
echo "Snapshot DEST: $DEST"

# 1. Idempotent cleanup of any prior containers
docker rm -f aic_eval aic_model 2>/dev/null || true

# 2. Bring up aic_eval headless (matches Phase 1 snapshot script's invocation)
echo "Starting aic_eval container (headless, ground_truth=true)..."
docker run -d --name aic_eval --gpus all --runtime=nvidia \
  -e NVIDIA_VISIBLE_DEVICES=all -e NVIDIA_DRIVER_CAPABILITIES=all \
  --net=host --privileged \
  ghcr.io/intrinsic-dev/aic/aic_eval:latest \
  ground_truth:=true start_aic_engine:=true gazebo_gui:=false

# 3. Wait for Gazebo + controller_manager to settle
sleep 35

# 4. Capture aic_controller-namespace topics (Open Q1 settlement)
echo "Capturing aic_controller-namespace topics..."
docker exec aic_eval bash -c '
  source /opt/ros/kilted/setup.bash &&
  ros2 topic list | grep -E "^/aic_controller|^/aic/gazebo|controller_state|joint_commands|pose_commands"
' | tee "$DEST/aic_controller_topic_list.txt"

# 5. Snapshot topic info (type + QoS) for each candidate topic
for topic in /aic_controller/joint_commands /aic_controller/pose_commands /aic_controller/controller_state /aic/gazebo/contacts/off_limit; do
  echo "--- $topic ---" >> "$DEST/aic_controller_topic_info.txt"
  docker exec aic_eval bash -c "source /opt/ros/kilted/setup.bash && ros2 topic info $topic --verbose 2>&1" \
    >> "$DEST/aic_controller_topic_info.txt" || true
done

# 6. Bring up aic_model (CheatCode policy) so off_limit contacts can fire
echo "Bringing up aic_model (CheatCode)..."
cd "$HOME/Documents/aic" && ./scripts/run_cheatcode.sh headless &
RUN_PID=$!
sleep 60   # wait for stack + first contact-relevant motion

# 7. Capture /aic/gazebo/contacts/off_limit for 120s (covers a full cheatcode cycle)
echo "Capturing /aic/gazebo/contacts/off_limit for 120s..."
docker exec aic_eval bash -c '
  source /opt/ros/kilted/setup.bash &&
  timeout 120 ros2 topic echo /aic/gazebo/contacts/off_limit
' | tee "$DEST/aic_eval_offlimit_capture.txt" || true

# 8. Tear down
echo "Tearing down..."
cd "$HOME/Documents/aic" && ./scripts/run_cheatcode.sh stop || true
docker rm -f aic_eval aic_model 2>/dev/null || true

echo "Snapshot complete. Files in $DEST:"
ls -la "$DEST"
```
