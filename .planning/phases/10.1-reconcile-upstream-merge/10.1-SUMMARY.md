## Critical infrastructure constraints (carried from Phase 11-01)

These were discovered through failure last session and apply to ANY further work on this codebase, including the merge resolution:

- **destroy_subscription/create_subscription is racy under MultiThreadedExecutor** → use idempotent helpers on `sub.topic_name`
- **`root.after()` from a daemon thread requires `_gui_ready` guard** → mirror `_append_log` pattern
- **Sub topics are sticky across sim ↔ real mode switches** → call `_*_ensure_*_topics` before refreshing
- **xdotool key without windowactivate fails silently** → activate first, sleep, then key

If upstream's `0944414` or `3c40417` reverts any of these patterns, that needs to be flagged during the diff inventory and resolved in our favor.
