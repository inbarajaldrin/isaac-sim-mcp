## Critical Anti-Patterns (do NOT repeat)

| Pattern | What happened | Severity | Prevention |
|---------|---------------|----------|------------|
| Mirror publishing to `/joint_commands` | Created 30 Hz feedback loop through `control_gui._ext_cmd_callback` → ~60 action goals/sec → executor saturated → `qs_*` services time out | blocking | Mirror MUST publish to `/joint_commands_lerobot` (or any non-`/joint_commands` topic). Documented in `linux-env/CLAUDE.md` CRITICAL section. |
| Duplicate USD references sharing inner prim name | Spawning two instances from `lego_red_2x3.usd` → both inner bodies named `red_2x3` → PhysX friction-patch explosion → quick_start hangs | blocking | Keep `LEGO_USDS` at one unique USD per instance. For "fewer legos" use spawn-skip (task 2 below), not duplicate refs. |
| `touch extension.py` mid-session | Kit's hot-reload watcher races with in-flight `quick_start` → stage half-mutates → Isaac Sim hangs | blocking | When changing module-level constants in extension.py: shut down Isaac Sim cleanly (`isaacsim_launch.sh close`), edit, then relaunch. Never touch mid-session. |
| Topic publish to `/arm_controller/joint_trajectory` | Under `mock_components/GenericSystem` (Linux), JointTrajectoryController only honors action goals — topic publishes are silently ignored | advisory | Use `/arm_controller/follow_joint_trajectory` action interface. control_gui already does this. |
| Class-level constants on `SOArm101ControlGUI` | `_hot_reload_logic` re-imports the module but instance's `__class__` still points to old class — new constants invisible | advisory | Put new constants at MODULE level (see `REC_LEGOS_BY_COLOR` etc near top of `control_gui.py`). |
| Forgetting to update `_hot_reload_gui` tab list | Adding a tab via the notebook builder alone won't make it appear after Ctrl+Shift+R — `_hot_reload_gui` at line 3055 has its own hardcoded tab list | advisory | Edit BOTH the notebook builder around line 1948 AND the rebuild loop in `_hot_reload_gui`. |
