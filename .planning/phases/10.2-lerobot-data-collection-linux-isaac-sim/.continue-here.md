<decisions_made>

- **Per-instance lego USDs over Cloner.replicate_physics**. Cloner
  fixed PhysX but ROS2PublishTransformTree disambiguated duplicate
  child_frame_ids with ugly path-prefixed names
  (`World_Objects_red_2x3_1_red_2x3`) — unusable by name-based
  recording loop. Per-instance USDs give truly unique inner prims,
  restore legacy `{wrapper}/{wrapper}` body-path convention.
- **Cup `cluster_offset_*` applied pre-`_to_world`** via
  `_apply_cup_cluster_offset` helper. randomize_cups validates in
  robot-relative space against lego/robot AABBs — post-`_to_world`
  offset would let validation pass on positions that overlap legos
  after the cluster shifts.
- **Publisher uses structural `_get_prim_path` fallback** (find
  first child with UsdPhysics.RigidBodyAPI). Legacy
  `{name}/{name}` produced empty target lists after the lego rename.
- **ArUco IDs: red=3, green=2, blue=1**. User's chosen convention.
  Drop wrappers auto-rename to drop_1/2/3 via
  `/World/Containers/drop_{aruco_id}`. Mirrored in 3 repos.
- **Workspace ground = two-layer split**. Main `/World/defaultGroundPlane`
  keeps neutral gray + collision; small `/World/workspace_ground`
  UsdGeom.Mesh on top is matte black for camera contrast. Objects
  beyond the tile still rest on the floor.
- **`CUP_LAYOUT_REAL_WORLD` separate from `CUP_LAYOUT_DEFAULTS`**.
  Two distinct intents: factory baseline vs physical-rig
  calibration. Match Real World is a partial override (only
  offset+gap), preserves mode/radius/angle/color_order.
- **Match Real World auto-called BEFORE `add_cups`** in quick_start.
  Cups don't exist yet on first quick_start — calling before ensures
  CUP_LAYOUT has calibrated values when add_cups reads it.
</decisions_made>

## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Cloner-with-shared-inner-names | Cloner.replicate_physics solves PhysX but leaves ROS2PublishTransformTree with duplicate child_frame_ids; the bridge then path-prefixes the disambiguating name (e.g. `World_Objects_red_2x3_1_red_2x3`) which breaks name-based downstream lookups. | advisory | When duplicating rigid-body USD references, use **per-instance USD copies on disk** with renamed inner prims, not Cloner. See `_get_or_make_lego_instance_usd` in extension.py. |
| Sdf.PrimSpec.name without path-rewrite | Renaming a prim spec via `name` setter doesn't rewrite material:binding targets or shader connection paths inside the same layer — bindings break and the asset loses its colors. | advisory | After rename, export to USDA text and run a regex `<(/old_name)(?=[/>])` → `</new_name` substitution before writing the final file. |
| Authoring on instance-proxy children | URDF importer makes per-link `visuals` Xform a USD instance; binding MaterialBindingAPI on its proxy children fails with "Cannot author to instance proxy". | advisory | Call `prim.SetInstanceable(False)` on the instance Xform before binding. With one robot in scene, this is essentially free. |
| Listbox height edits via hot_reload | tk Listbox `height` is a creation-time setting. Hot-reload re-binds methods but doesn't recreate widgets. | advisory | Document that listbox-shape changes need a control_gui restart. |
| Post-_to_world offset on validated layouts | randomize_cups validates positions in robot-relative space; applying a world-frame offset AFTER _to_world skips validation against shifted positions. | advisory | Apply via `_apply_cup_cluster_offset` BEFORE `_to_world` in all 3 callsites. |
