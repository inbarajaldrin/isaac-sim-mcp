# Pause Note — Sim/Real Dataset Parity (Phase 10.2)

## Critical Anti-Patterns (do NOT repeat these)

| Pattern | Description | Severity | Prevention Mechanism |
|---|---|---|---|
| Custom rerun loader for lerobot datasets | Wrote `/tmp/review_episodes_rerun.py` using `rr.AssetVideo` after seeing the canonical `lerobot-dataset-viz` line in CLAUDE.md and ignoring it. AssetVideo path fails on Ubuntu 22.04 because system ffmpeg 4.4.2 < rerun's required 5.1, sending us down a 100-line cv2-frame-by-frame fallback that re-implemented `lerobot-dataset-viz`. | blocking | Always use `pixi run --manifest-path …/linux-env/pixi.toml lerobot-dataset-viz --repo-id local/<name> --root <path> --episode-index N --mode local`. Re-invoke with different `--episode-index` to add to the recordings dropdown. Documented in `linux-env/CLAUDE.md` gotchas. |
| Mirror /joint_states as the action source | Original `joint_states_to_commands.py` published `/joint_states` verbatim → action == state to 14 decimals, useless for BC. | blocking | New mirror reads `/{arm,gripper}_controller/controller_state.reference.positions` (JTC's per-tick commanded setpoint). Filename is historical — *reads controller reference, not joint_states*. |
| Sending Left key during lerobot's Phase B | When the next pick-place fails fast (~14s after Right), the orchestrator's Left signal arrived during lerobot's reset_time_s window and made the post-Phase-B rerecord-check clear the buffer holding the *previous* successful save. ep6 was lost this way. | blocking | `_rec_count_phase_a_starts()` + `_rec_wait_for_new_phase_a()` gate the next pick-place on lerobot logging a *new* "Recording episode N+1" line. Right/Left always land in the intended Phase A. |
| SIGINT during save_episode | Orchestrator broke loop on `success_count >= n_target` and immediately SIGINT'd lerobot, killing it mid-`save_episode()` for the last cycle. | advisory | 20s drain window before stop (longer than reset_time_s=15s) so the last save finishes. |
| `'\\n'.join(msg.split('\\n'))` in Python source | The literal `'\\n'` in source is the *two-character* string `\n` (backslash+n), NOT a newline. `_rec_run_one_qs_cycle`'s sentinel detection used this on `log_resp.message` (which has REAL newlines), so split returned a 1-element list and the success sentinel never matched — every passing cycle got mis-classified FAIL after 90s timeout. | advisory | Use `'\n'` (single character). Fixed in commit `8503359`. |

## Pre-Execution Critique Required

Before fixing audit item 4 (URDF joint units), verify the hypothesis with a small diagnostic:

```bash
# Compare joint position over the same physical pose between sim and real
# Use the gripper at "fully open" — what value does each report?
# Sim:  ros2 service call /so_arm101_control_gui/gripper_open std_srvs/srv/Trigger
#       ros2 topic echo /joint_states --once  | grep -A1 gripper
# Real: same, on real hardware
# If sim reports ~6.5° and real reports ~29°, ratio ≈ 4.5 — likely a gear ratio.
```

The fix is one of:
- **(a)** Multiply sim gripper output by the gear ratio in the action mirror (post-process). Cheapest.
- **(b)** Update the sim URDF gripper joint to use the same convention as Feetech (ground-truth fix). Bigger blast radius.
- **(c)** Document the difference and accept calibration in the BC training pipeline.

Decide which before patching code.
