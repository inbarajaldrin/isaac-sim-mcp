<task id="02-02">
<title>Transform each widget call-site to use the matching factory</title>
<read_first>
- /tmp/07.1-migration-plan.md (the checklist from 02-01)
- ~/Projects/Exploring-VLAs/vla_SO-ARM101/src/so_arm101_control/so_arm101_control/control_gui.py — around each line from the inventory

</read_first>
<action>
For each row in the migration plan, apply the appropriate transformation. General patterns:

**Spinbox (no label kwarg needed if explicit):**
```python
# Before
tk.Spinbox(parent, textvariable=self._foo_var, from_=0.0, to=1.0,
           increment=0.1, width=5)
# After
self._register_spinbox(parent, label='Foo', tab='IK', section='Target',
                       textvariable=self._foo_var, from_=0.0, to=1.0,
                       increment=0.1, width=5)
```

**Checkbutton (text= becomes label=):**
```python
# Before
tk.Checkbutton(parent, text='Ground Plane', variable=self._var,
               command=self._cmd_toggle_ground_plane)
# After
self._register_check(parent, label='Ground Plane', section='Scene',
                     variable=self._var,
                     command=self._cmd_toggle_ground_plane)
```

**Entry:**
```python
# Before
tk.Entry(topic_row, textvariable=self._grasp_topic_var)
# After
self._register_entry(topic_row, label='Grasp Topic', tab='Grasp', section='Topic',
                     textvariable=self._grasp_topic_var)
```

**Listbox:**
```python
# Before
self._drop_listbox = tk.Listbox(drop_list_frame, height=4, ...)
# After
self._drop_listbox = self._register_listbox(
    drop_list_frame, label='Drop Targets', tab='Grasp', section='Drop Targets',
    height=4, ...)
```

**Scale (joint sliders):**
```python
# Before
slider = tk.Scale(row, variable=var, from_=lo, to=hi, ...)
# After — use the joint name as the label (matches the tk.Label to the left)
slider = self._register_scale(row, label=name, tab='FK', section='Joint Sliders',
                              variable=var, from_=lo, to=hi, ...)
```

**Notebook — register post-creation** (constructor signature stays tk-native):
```python
notebook = ttk.Notebook(root)
self._register_notebook(notebook, label='tab')
```

**Log Text widgets** — register post-creation:
```python
self._process_log = tk.Text(proc_frame, ...)
self._register_log_text(self._process_log, label='process_log')
# same for self._error_log
```

**Label heuristic for paired widgets:** When a Spinbox/Entry/Scale has a `tk.Label` in the same parent Frame immediately before it, the Label's text IS the widget's label. If the Label has a trailing colon (e.g. `'X:'`, `'Duration (s):'`), keep the colon in the widget ID — it matches what the user sees on screen. Examples:
- `X:` → `label='X:'` (the colon is part of the ID)
- `Approach height (m):` → `label='Approach height (m):'`

**At `_setup_gui_thread` completion**, call `self._finalize_widget_registry()` to resolve label collisions (D-04). Best place: the very end of `_setup_gui_thread` after all `_build_*_tab` calls return.

Work through the migration plan row-by-row. After each row, re-run the Plan 07-01 button test to catch any accidental Button regression:
```bash
python3 -m pytest src/so_arm101_control/so_arm101_control/test_button_service_mapping.py -v
```
