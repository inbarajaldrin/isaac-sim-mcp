<objective>
Create a CI gate that prevents future direct widget constructions from slipping in unnoticed. Mirror Plan 07-01 Task 01-03's AST-test pattern, but extended to the five new interactive widget types. Implements CONTEXT.md decision D-03 and ROADMAP success criterion #4.

**Deliverable:** a new test file `test_widget_registry.py` (or extension of the existing `test_button_service_mapping.py`) that fails when:
- A direct `tk.Spinbox` / `ttk.Spinbox` / `tk.Checkbutton` / `tk.Entry` / `ttk.Entry` / `tk.Listbox` / `tk.Scale` / `ttk.Scale` constructor is called outside the factory method's own body.
- A `self._register_<type>(...)` call is missing its required `label=` kwarg.
- A `label=` kwarg value is not a string literal (dynamic label expressions make IDs unstable).
</objective>

<action>
Create `test_widget_registry.py` modeled on `test_button_service_mapping.py`. Structure:

1. **Module-level constants:**
   ```python
   FACTORY_TO_FORBIDDEN = {
       '_register_spinbox':   {('tk', 'Spinbox'), ('ttk', 'Spinbox')},
       '_register_check':     {('tk', 'Checkbutton')},
       '_register_entry':     {('tk', 'Entry'), ('ttk', 'Entry')},
       '_register_listbox':   {('tk', 'Listbox')},
       '_register_scale':     {('tk', 'Scale'), ('ttk', 'Scale')},
   }
   ```
   (Notebook + Text widgets are registered post-construction — the tree walk for those is different; see task 03-02.)

2. **Test `test_no_direct_interactive_widget_construction`:**
   - Parse `control_gui.py` AST.
   - For each factory in `FACTORY_TO_FORBIDDEN`, find its function definition; record the set of AST node ids inside that factory's body.
   - Walk all `ast.Call` nodes. For each call whose function is `ast.Attribute(value=ast.Name('tk' or 'ttk'), attr='Spinbox/Checkbutton/Entry/Listbox/Scale')`:
     - If the call's `id(node)` is inside one of the factory bodies → ALLOWED (the factory itself constructs the real widget).
     - Otherwise → FORBIDDEN. Append `(lineno, f'tk.{attr}')` to a failure list.
   - Assert failure list is empty with a clear message naming every offender.

3. **Test `test_factory_calls_require_label`:**
   - For each `self._register_<type>(...)` call, check that keyword args include `label=`.
   - Assert no missing `label=` kwargs.

4. **Test `test_factory_labels_are_string_literals`:**
   - For each `self._register_<type>(...)` call, the `label=` kwarg's value must be an `ast.Constant` of type `str`. Dynamic labels break the ID stability.
   - Assert all labels are literals.

5. **Test `test_register_notebook_and_log_text_present`:**
   - Assert `_register_notebook` is called at least once (the main notebook).
   - Assert `_register_log_text` is called at least twice (process_log + error_log).

6. **Sanity `test_factory_count`:**
   - Assert at least 10 `_register_spinbox`, 5 `_register_check`, 3 `_register_entry`, 2 `_register_listbox`, 6 `_register_scale` calls. Numbers match Plan 07.1-02 acceptance criteria.

7. **`if __name__ == '__main__'` block** running all tests, same pattern as `test_button_service_mapping.py`.
</action>

<title>Regression-test the test</title>

<action>
For each of the five forbidden-construction types, inject a deliberate violation and verify the test catches it. Pattern:

1. Copy `control_gui.py` to `/tmp/control_gui.good.py` (backup).
2. For each of Spinbox, Checkbutton, Entry, Listbox, Scale:
   - Revert ONE factory call to its forbidden equivalent (e.g., replace `self._register_spinbox(parent, label='X:', ...)` with `tk.Spinbox(parent, ...)` — drop the label for extra coverage).
   - Run `python3 src/so_arm101_control/so_arm101_control/test_widget_registry.py`.
   - Assert exit code is non-zero AND stderr mentions the specific widget type and line number.
   - Restore from backup.
3. ALSO test: a `self._register_spinbox(parent, textvariable=..., from_=0, to=1)` WITHOUT a `label=` kwarg — must fail with "label= required".
4. ALSO test: a `self._register_spinbox(parent, label=some_dynamic_var, ...)` — must fail with "label must be a string literal".

Document each injected-and-caught violation in a short comment at the top of a new file `/tmp/07.1-03-regression-evidence.md`. Delete the evidence file after human review.
</action>
