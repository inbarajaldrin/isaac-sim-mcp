<objective>
Replace the Real Test tab's "loop over all cached legos" Run handler with a color-dropdown-driven single-pick workflow. User picks a color (red / green / blue) from a dropdown that lists only colors with both a cached lego AND a cached cup, presses Run, and the arm executes one full _QS_SEQUENCE for the closest lego of that color. On success, evict the dropped lego from cache; on failure, leave cache untouched. The dropdown auto-updates on every cache mutation (refresh, clear, evict).

Locked design decisions live in `11-CONTEXT.md` D-01 through D-10. This plan implements them.
</objective>

<title>Implement `_cmd_real_run_one_color` (single-pick handler)</title>
<read_first>

<action>
Replace `_cmd_real_run_pick_drop` and `_real_pick_drop_thread` with the single-pick path. Steps:

1. Mark the OLD methods deprecated: keep their bodies (commented header `# DEPRECATED 11-01: replaced by _cmd_real_run_one_color, kept for reference for one cycle`) OR delete outright. **Delete** is cleaner per project conventions (no dead-code comments). Choose delete.

2. Add `_cmd_real_run_one_color(self)`:
   - `_ensure_real_state()` first
   - Read `color = self._real_selected_color_var.get()`. If empty, log `'Real: no color selected'` and return.
   - Pre-flight (mirror existing checks):
     - `_cached_cup_poses` non-empty
     - `_cached_lego_poses` non-empty
     - `self._qs_state != 'running'`
   - Spawn `threading.Thread(target=self._real_run_one_color_thread, args=(color,), daemon=True).start()`

3. Add `_real_run_one_color_thread(self, color)`:
   - Compute candidate legos: `[n for n in self._cached_lego_poses if n.split('_', 1)[0].lower() == color]`. If empty (race against eviction), log error + status='Error', return.
   - Pick lego = the candidate with smallest `sqrt(x² + y²)` from the cached pose, alphabetical tiebreak. Use a single sort key: `(math.hypot(p['x'], p['y']), name)`.
   - Compute `cup_name = _qs_auto_drop_for_lego(lego)`. Verify `cup_name in self._cached_cup_poses` — if not, log error and return (this should never trigger if dropdown filtering works, but defend in depth).
   - Re-inject cached poses (mirror existing per-cycle pattern at L4401–L4406):
     ```python
     with self._drop_lock:
         self._drop_data[cup_name] = dict(self._cached_cup_poses[cup_name])
     with self.objects_lock:
         self.objects_data[lego] = dict(self._cached_lego_poses[lego])
     ```
   - Populate listboxes via `self.root.after(0, ...)` for `_populate_object_list`, `_populate_drop_list`, `_qs_refresh_objects`. Sleep 0.6s for tk to flush.
   - `_qs_select_lego(lego)`. If False, log error + return.
   - Reset `_qs_abort_evt.clear()`, `_qs_resume_evt.set()`, `_qs_state = 'running'`.
   - Set status: `self._real_set_status(f'Running ({color})', step=f'preparing — {lego} → {cup_name}')`.
   - Loop over `_QS_SEQUENCE`. Per step: honor abort, call `_qs_execute_step`, on failure set `ok_all = False` and break. Status updates per step mirror the existing loop's format.
   - On `ok_all`:
     - `self._real_set_status('Complete', step=f'✓ {lego} → {cup_name}')`
     - Append log: `f'Real: cycle complete — {lego} dropped in {cup_name}'`
     - Call `self._real_evict_lego_from_cache(lego)` (task 11-01-05)
   - On not `ok_all`:
     - `self._real_set_status('Halted', step=f'✗ {lego} at "{label}"')`
     - NO eviction (D-06)
   - `finally: self._qs_state = 'idle'`

4. Explicit log line at thread start:
   `'Real: single-pick mode — using cache only, no live topic source'`
   This satisfies D-07's explicit-isolation log marker.
</action>

7. Log: `f'Real: evicted {lego_name} from cache (remaining: {len(self._cached_lego_poses)})'`.
