## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Class-level constants don't survive `_patch_methods` hot-reload | Class attrs like `_REAL_COLOR_TO_CUP`, `_DROP_SCAN_*` are not copied during hot-reload — patched methods accessing them get silent `AttributeError` in daemon thread. | advisory | Default to **module-level** constants in this codebase (above the class). The hot-reload comment at L89-90 of control_gui.py is load-bearing. |
| Verify-before-claim-complete | Asserting "works" from grep + syntax + service-registered without invoking the service. | blocking | After every implementation, **invoke the service/button at least once** and check log/state. |
| Daemon threads swallow exceptions silently | `threading.Thread(daemon=True)` worker bodies don't surface exceptions — failures look like "did nothing". | advisory | Wrap every `_real_*_thread` body in `try/except Exception as e: self._append_log(repr(e), 'err')`. |
| Two-source-of-truth state divergence | Local cache vs remote authority diverge; "skip if cache matches" hides drift. | advisory | Default to "always re-query authoritative" rather than "check local cache then skip". The skip saves microseconds and hides bugs. (D-13: StringVar vs sub. D-14: tracker vs planning scene.) |
| `TRANSIENT_LOCAL --once` returns stale retained, not latest | `ros2 topic echo --once` on a TRANSIENT_LOCAL topic returns an old retained message from the depth buffer, not the most recent publish. | advisory | Use `--qos-durability volatile` when debugging "what was JUST published". |
| SO-ARM101 weak drives can't sustain >30°/s | stiffness=15, damping=0.15 — trajectories faster than ~30°/s sustained stall the controller and `_send_arm_goal(blocking=True)` hangs. | advisory | Compute duration from `max_joint_delta / 30°/s` with floor 1.5s. |
