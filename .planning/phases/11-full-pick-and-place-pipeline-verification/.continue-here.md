# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure this session.
> Acknowledge each one explicitly before proceeding.

- [ ] **CONSTRAINT — destroy_subscription/create_subscription is racy under MultiThreadedExecutor.** Any sub-replacement helper called from a daemon thread (not the executor's worker threads) WILL eventually crash with `InvalidHandle: destruction was requested`. The mitigation is structural: make sub-replacement functions **idempotent on `sub.topic_name`** so the destroy/create race window collapses from once-per-button-press to once-per-process. Don't try to "fix" the race with locking — the executor doesn't expose its internal sub-list lock.

- [ ] **CONSTRAINT — `root.after()` from a daemon thread requires `_gui_ready` guard.** Without the guard, any thread-ish helper that schedules tk updates will raise `RuntimeError: main thread is not in main loop` on any error path during shutdown or before mainloop fully starts. Mirrors the pattern `_append_log` already uses. Wrap in `try/except RuntimeError` as belt-and-suspenders.

- [ ] **CONSTRAINT — Sub topics are sticky across sim ↔ real mode switches.** Any helper that "refreshes" data MUST first call the matching `_*_ensure_*_topics` (sim or real) to assert which topology it expects. Without this, `_cmd_grasp_refresh` after a Real Test session silently leaks `/objects_poses_real` data into the sim collision scene.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention |
|---------|-------------|----------|------------|
| destroy_subscription on daemon thread | Race against MultiThreadedExecutor → InvalidHandle | blocking | Idempotent helpers on `sub.topic_name` |
| `root.after()` from daemon thread | tkinter "main thread is not in main loop" on any error path | blocking | `_gui_ready` guard + try/except RuntimeError |
| Sticky sub topics across modes | Sim refresh pulls from /*_real after Real Test session | blocking | `_*_ensure_*_topics` helpers, called from refresh + clear paths |
| visited_ids only added on success | Flaky cup pulls sweep back forever | advisory | Mark visited on every exit path |
| `xdotool key` without `windowactivate` | Keys go to focused (wrong) window, hot-reload fails silently | advisory | Always activate target window first |
| Math-before-experiment skipped | Wasted debug cycles on wrong hypothesis (parallax) | advisory | Do order-of-magnitude math BEFORE proposing geometric fixes |
