## Tasks

### Task 1: PARITY-09 motion deficit hunt (≥95% commanded amplitude)

**Current state:** post-fix `b682de1`, round-trip on shoulder_lift_joint shows Δ≈0.016 rad on +0.10 rad command (16% effective). Writes reach the articulation; remaining 84% is dissipated somewhere.

**Hypothesis order to investigate:**
1. **OGN articulation_controller PD residual** — Phase 1 setup_action_graph wires an OGN ArticulationController node at `/Graph/ActionGraph_UR5e/articulation_controller` subscribed to `/joint_states`. Even with the explicit "deactivated" log line in `_setup_kinematics`, the node may still tick — verify via `og.Controller.attribute("inputs:enabled", node).get()`.
2. **PD stiffness vs cable mass-coupling** — articulation now has 46 DOFs (6 arm + 40 cable D6). The arm joints' stiffness (~95-1322 per the today's audit) may be insufficient against the cable's coupled inertia. Test: temporarily detach cable (SetActive(False) on /World/UR5e/cable), re-run round-trip, compare.
3. **set_joint_positions vs apply_action(PD target) interaction** — the current code does both per tick (set_joint_positions for instant motion + apply_action for PD target refresh). The PD target may be stale by one tick, dragging the position back. Test: log `_articulation._physics_view.get_dof_position_targets()` post-write to compare commanded vs actual.

**Acceptance:** shoulder_lift_joint round-trip on +0.10 rad command produces Δ ≥ 0.095 rad within 2 seconds. Document the root cause + fix in commit message.
