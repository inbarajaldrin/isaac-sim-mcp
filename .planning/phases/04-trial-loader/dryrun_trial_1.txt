**kilted ↔ humble fastrtps_cpp message-type-hash incompatibility for stock
ROS messages (rosgraph_msgs/Clock, sensor_msgs/JointState, etc.).**

A2=PASS in 04-01-SUMMARY only verified `aic_control_interfaces/JointMotionUpdate`,
which was custom-built in the same workspace (kilted aic_eval) so the ABI
matches by construction — but did NOT verify stock messages whose type hashes
differ between kilted's humble's rosidl_generator outputs.

aic_engine subscribes to /clock as `rosgraph_msgs::msg::Clock` (kilted-side
type hash); Isaac Sim publishes it via the kit ROS bridge using the humble
py311 workspace's type hash. fastrtps's strict type-hash validation in
recent versions silently drops cross-distro messages.

The host-side subscriber WORKED earlier because the host's
`/opt/ros/humble/bin/ros2` uses humble type hashes that DO match
Isaac Sim's humble bridge.

### Why A2's "PASS" wasn't sufficient
04-01-SUMMARY A2 line: "kilted-built ABI loads against humble" — this only
verified the rosidl_typesupport_c symbol resolution (`UnsupportedTypeSupport`
import gate). It did NOT verify wire-level message receipt across the
distro boundary.
