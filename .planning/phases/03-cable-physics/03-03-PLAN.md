# Plan 03-03: SCENE-02 cable type variants + SCENE-03 gripper attach

## Objective

Two related deliverables on the load_robot surface:

- **SCENE-02:** `cable_type` parameter on `load_robot` selects between `sfp_sc_cable` (default) and `sfp_sc_cable_reversed`. Both use the same vendored cable USD; `_reversed` adds π to `cable_yaw`.
- **SCENE-03:** `attach_cable_to_gripper` bool parameter — when True, author a FixedJoint between the gripper finger link and the cable's plug-end link. New MCP atom `attach_cable_to_gripper` (4 surfaces per DX-02) for runtime invocation.

## Tasks

### Task 1 — Extend load_robot signature

Add to `load_robot(...)`:
```python
cable_type: str = "sfp_sc_cable",
attach_cable_to_gripper: bool = False,
gripper_initial_pos: float = 0.00655,  # Gazebo default; pass 0.0073 for sfp_sc_cable
```

Wire through `_cmd_load_robot` (handler) + `MCP_TOOL_REGISTRY['load_robot']` (param spec).

### Task 2 — Implement cable_type variants

In `load_robot`, after computing `cable_yaw`:
```python
if cable_type == "sfp_sc_cable_reversed":
    cable_yaw = cable_yaw + math.pi  # 180° flip
elif cable_type != "sfp_sc_cable":
    print(f"[AIC-DT] Unknown cable_type {cable_type!r}; defaulting to sfp_sc_cable")
```

### Task 3 — New MCP atom: attach_cable_to_gripper (4 surfaces per DX-02)

Add to:
1. `MCP_TOOL_REGISTRY['attach_cable_to_gripper']` with `prim_paths` (auto-detected) + `gripper_initial_pos` params
2. Handler map: `'attach_cable_to_gripper': '_cmd_attach_cable_to_gripper'`
3. `_cmd_attach_cable_to_gripper` method that:
   - Locates the plug-end link prim (use Plan 03-01's discovery — typically `/World/UR5e/cable/sc_plug_link`)
   - Locates the gripper finger prim (`/World/UR5e/aic_unified_robot/gripper_hande_finger_link_l` per CLAUDE.md)
   - Authors a `UsdPhysics.FixedJoint` at `<gripper_finger>/CableAttachJoint` connecting body0 + body1
   - Idempotent: removes prior CableAttachJoint before creating new one
4. UI button in extension.py panel

### Task 4 — quick_start integration

If `attach_cable_to_gripper=True` was passed to load_robot, automatically invoke the new atom after `load_robot` returns. Update quick_start chain to pass `attach_cable_to_gripper=True` by default (per Claude's discretion in CONTEXT D-03).
