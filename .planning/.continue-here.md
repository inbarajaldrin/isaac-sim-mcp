## Critical Anti-Patterns (discovered this session)

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Declaring failure "pre-existing" without back-to-back baseline | After Phase 7 refactor, the first `grasp_home` returned "Planning failed (error -2)". I dismissed it as edge-case flakiness without running HEAD for comparison. User pushed back; HEAD comparison revealed identical flakiness rate (not a regression). | advisory | Always run N ≥ 3 cycles of HEAD and refactor side-by-side before attributing failure to either side. |
| `ros2 param set` type auto-inference on string params | `ros2 param set node widget_value 0.20` silently coerces to float type, then the service reads the old STRING value. Confusing for agent flows. | advisory | Always wrap set-values in single quotes when targeting string-typed params: `ros2 param set node widget_value "'0.20'"`. Documented in `07.1-AGENT-DRIVEN-IK-DEMO.md`. |
| tkinter mainloop ignores SIGINT/SIGTERM | orphaned `control_gui` python process after launch-parent died didn't respond to graceful signals due to tkinter mainloop. | advisory | Use `xdotool search --name "SO-ARM101" windowclose` to trigger WM_DELETE_WINDOW instead of kill -SIGTERM. If no window owner (orphaned), SIGKILL is safe. |
| `gsd-read-guard.js` hook firing despite CLAUDECODE env var | PreToolUse hook fires on every Edit/Write even when `CLAUDECODE=1` is set. Claude Code spawns hooks in a stripped env. Doubles tool-turn cost. | advisory | Removed from `~/.claude/settings.json` during this session. Backup at `~/.claude/settings.json.bak.20260418-214913`. If it re-appears after a `/gsd-update`, remove again. |
