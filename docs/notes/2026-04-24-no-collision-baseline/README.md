# No-Collision Baseline — 2026-04-24 post-restart batch

**Captured:** 2026-04-24, motion timestamps 07:08–07:10 UTC
**Why preserved:** First batch of post-restart motions where **no cup contact occurred** despite the underlying tracking-lag mechanism still being present (RTF ~0.16, wrist_flex peak lag 10–13°). Useful as a positive baseline for future comparisons — same lag magnitude as the pre-restart `20260423T225859_drop_sweep` baseline but different geometric outcome.

## Session context (what was happening when these were captured)

1. Had been investigating SO-ARM101 wrist_flex tracking lag (13° baseline → cup collisions)
2. Tried damping bump 0.15 → 1.5 — made tracking WORSE (reverted)
3. Discovered all `use_sim_time=False` while `/clock` runs degraded → forum #309047 root-cause pattern
4. Flipped use_sim_time:=true on 9 nodes via `ros2 param set` — broke RViz (TF buffer corruption from runtime flip)
5. **Restarted control stack** (this wiped use_sim_time back to launch defaults = False)
6. **Ran these 10 motions immediately after restart** — none knocked any cup

## What this batch DOESN'T prove

- ❌ Tracking lag is fixed (it's not — same 10-13° magnitude as before)
- ❌ RTF is fixed (still 0.15–0.17, target 1.0)
- ❌ Cup collisions are eliminated (only 2 drop_sweeps sampled; forum #363351 pattern is ~1/5 collision rate)

## What this batch IS useful for

- ✅ Reference data for "system in current state, this is what no-collision motions look like"
- ✅ Cross-comparison with future drive-tuner experiments (we can re-run these exact planner trajectories via `--ref` once the drive tuner exists)
- ✅ Demonstrates planner is producing geometrically marginal trajectories (~8 mm clearance to cups when "successful") — argues for raising MoveIt's default_robot_link_padding rather than tuning drive gains
- ✅ Captures full-stack plot showing `fb_pos`/`/joint_states` track `ref_pos` exactly (mock-hardware echo) while `isaac_pos` lags — proves ros2_control's tolerance check is blind
- ✅ The grasp_home outliers (43° and 104° differential lag) deserve investigation — those would have produced collisions if the path went near a cup
